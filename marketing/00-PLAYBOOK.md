## ⚠️ 먼저 처리해야 하는 것 (캠페인 시작 전)

**법무 上 리스크 2건 — 리드 문자 보내기 전에 반드시:**
1. **리드 팝업에 동의 문구가 없음** — 전화번호를 받으면서 개인정보 수집·광고성 수신 동의를 안 받고 있음. 이 상태로 `buildLeadReminder` 문자를 보내면 정통망법 위반 소지 (과태료 최대 3천만원). → LeadPopup.tsx 코드 수정 (즉시 가능)
2. **리드 리마인드·재참여 문안에 (광고) 표기·수신거부 안내 없음** — notice.ts 문안 수정 + 21시~08시 발송 금지 준수
3. (중) 노쇼 정의 명문화 — "몇 시까지 취소하면 이월인지"가 어디에도 없음. "전날까지 취소 → 이월, **건강 사유는 당일도 소멸 없음**" 권장 (안전 의무와 충돌 방지)
4. (중) 보증금 수취 계좌를 개인 → 사업자 명의로 (BANK 상수 1곳 수정)

**측정 선행 1건**: `?apply=1` 처리 시 replaceState가 UTM을 지울 가능성 — 배포 후 시크릿창에서 UTM 링크 접속 → GA4 실시간에 소스 찍히는지 5분 검증. 안 찍히면 apply 파라미터만 제거하도록 수정 필요.

1. **"100명 목표" 공개 여부** — 랜딩 FAQ에서는 빼셨는데, 스레드 전략(빌드인퍼블릭)과 콘텐츠 일부는 목표 공개가 축입니다. 권장: **랜딩·인스타(서비스 채널)에서는 비공개, 스레드(개인 채널)에서만 공개** — 서비스는 담백하게, 개인 실험기는 투명하게. 03 파일 캡션 ①의 "100명" 문장도 이 결정에 맞춰 조정.
2. **당일 취소 규칙** — "전날 23:59까지 취소=이월 / 이후·노쇼=소멸(첫 1회 유예) / 건강 사유=항상 이월" 안을 약관에 명문화할지
