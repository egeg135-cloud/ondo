## 🎯 총평

**신뢰 구조는 초기 서비스치고 매우 탄탄하다.** 후기 3종(이미지+DB 실시간+텍스트), 신청자 구성 차트, 비교표, "하는 것/안 하는 것" 투명성, 매칭 실패 환불 보장, GEO 정의문, GA4 이벤트 14종, 공유카드, 리드팝업 — 인프라의 8부 능선은 이미 지어져 있다.

**문제는 3가지 축이다:**
1. **신뢰를 스스로 갉아먹는 가짜 신호** — 하드코딩된 잔여석, 동일한 A/B 변형, 죽은 링크 4개
2. **검색·AI에게 보이지 않는 페이지** — SPA 빈 껍데기 + canonical이 Netlify를 가리킴 + 7.7MB 이미지
3. **돈이 새는 두 구멍** — 신청→입금(수동 계좌이체) 이탈과 체험→멤버십 전환 플로우 부재

## 1️⃣ CRO / 전환 분석

### 심각도순 이슈

1. **[치명] 3초 만에 뜨는 전화번호 팝업** (`App.tsx:49-56`) — 히어로를 읽기도 전에 전면 모달 + `autoFocus`로 모바일 키보드까지 즉시 등장. **exit intent 감지 코드가 이미 있는데(`App.tsx:74-85`) 추적만 하고 팝업에 안 씀** — `onExit`에서 띄우도록 변경, autoFocus 제거.
2. **[높음] 결제 마찰** — 신청 완료 후 수동 계좌이체(개인명 예금주)가 최대 이탈 구간. 성공 오버레이의 주황색 주CTA가 "📸 스토리에 자랑하기"인데 이 시점 유일한 목표는 **입금**. 단기: 복사 버튼+카카오뱅크 딥링크. 중기: 토스페이먼츠/카카오페이 링크 결제.
3. **[중간] 가격 카드 추천 신호 충돌** — 시각 위계는 체험권(주황 테두리), 언어는 멤버십("👑 추천"). 전략 하나로 정렬 필요.

### 신청 플로우 (ApplyModal 5단계)
- **Step 1 필드 과다**: 나이차 매칭 범위(±N세)는 99%가 기본값 → 폼에서 제거, MBTI는 후반/신청 후로 이연
- **페이스 중복 질문**: PaceQuickModal에서 이미 골랐는데 Step 2에서 재노출 → "6분 페이스로 신청 중 · 변경" 한 줄로 접기
- **마케팅 동의 재확인 인터럽트**: 최종 제출 클릭 후 확인 모달이 끼어들어 제출 실패로 오인한 이탈 위험 → 성공 오버레이 이후로 이동
- 5km 불가 선택 시 안내에 "매칭 안 되면 전액 환불돼요" 명시로 불안 상쇄

### A/B 테스트 가설 (ICE 순)
| 가설 | ICE |
|---|---|
| LeadPopup 트리거 3초 → exit intent | 8.3 |
| PaceQuickModal CTA "결제하기" → "이번 주 자리 신청하기" | 8.0 |
| 신청 폼 5→3단계 축소 | 6.7 |
| 히어로 pain형 vs 결과형("4주 뒤, 목요일마다 뛰는 사람이 됩니다") | 6.7 |
| 가격 카드 체험권 강조 vs 멤버십 강조 | 6.0 |

## 2️⃣ 카피 / 오퍼 분석

### 히어로 진단
- 잘됨: pain-first 헤드라인이 실제 후기 언어와 일치(VOC 미러링), 구체성 3종(목요일/3~5명/여의도 5km), 가격 박힌 CTA
- 문제: 헤드라인이 문제만 말하고 해결 약속이 없음. 차별화 축("크루가 아니라 매칭")이 스크롤 한참 아래 비교표에만 존재

### 카피 Before → After (대표 5개)
1. **히어로**: "혼자서는 자꾸 미루게 되는 퇴근 후 러닝." → "혼자서는 자꾸 미루게 되는 퇴근 후 러닝, **목요일마다 기다리는 사람이 생기면 달라집니다.**"
2. **메인 CTA**: "이번 주 1회 체험 5,000원" → "**이번 주 목요일, 5,000원으로 체험하기**" (동사 추가)
3. **가격 헤드**: "눈치도 뒤풀이도 없이, 매칭에만 집중한 가격" → "**커피 한 잔 값으로, 목요일마다 기다리는 러닝메이트**" (멘탈 어카운팅)
4. **멤버십 카드**: → "**회당 4,750원** — 1회 체험보다 저렴하게, 4주간 10~15명의 새 러닝메이트를 만나요" (단가 분해 + 폼 안에 숨은 구체 수치 승격)
5. **신청 완료**: "아래 계좌로 입금하시면 매칭이 시작돼요" → "**10분 안에 입금하시면 이번 주 매칭 명단에 바로 올라가요. 매칭 확정은 수요일 카톡으로.**" (자이가르닉 — 미완료 루프 닫기)

⚠️ 워딩 불일치: 가격 카드는 "러너 2~4명", 나머지 전체는 "3~5명" — 통일 필요.

### 오퍼 구조 강화안
- **보증에 이름 붙여 승격**: "면죄부"가 커밋 메시지에만 있고 현재 FAQ 12번째에 묻힘 → **"첫 러닝 보증"**(매칭 안 되면 전액 환불 + 첫 체험 불만족 시 환불)으로 재포장해 가격 카드 바로 아래 + CTA 옆 배치. 리스크는 인당 5,000원뿐.
- **체험→멤버십 브릿지 오퍼**: "체험 후 48시간 내 등록 시 체험비 5,000원 차감(=14,000원)" — 체험비가 매몰비용이 아닌 크레딧이 되어 손실회피가 전환 방향으로 작동
- **비용 0원 보너스 스택**: "여의도 5km 코스맵 + 오버페이스 방지 가이드"(RELATED_ARTICLES 스텁 콘텐츠)를 신청자 전원 카톡 보너스로 — 죽은 링크 문제도 동시 해결

### 심리 원칙: 누락 항목
진짜 희소성(실데이터), 앵커링(`originalPrice` 필드 있는데 미사용), 권위(운영자/매칭 근거 소개 0), 수치형 밴드왜건("지금까지 N명이 함께 뛰었어요" 부재), 상호성(무료 기브 없음)

## 3️⃣ SEO / AI 검색 분석

### 치명 이슈
1. **canonical → Netlify** (위 즉시수정 #1)
2. **robots.txt/sitemap.xml 부재** — `public/`에 파일 추가 (Vercel은 실파일이 rewrite보다 우선)
3. **SPA 빈 껍데기** — body가 `<div id="root"></div>`뿐. 구글은 늦게라도 렌더링하지만 **네이버(Yeti)와 AI 크롤러(GPTBot, PerplexityBot)는 빈 페이지로 봄**. FAQ 12개, 정의문, 가격이 전부 안 보임 → 프리렌더링(vite-plugin-prerender 또는 빌드 후 puppeteer 스냅샷)이 SEO/AI SEO 전체의 전제 조건
4. **JSON-LD가 클라이언트 렌더** (`SeoJsonLd.tsx`) → `index.html` head에 정적 이동. **Event 스키마(매주 목요일 20시 여의도, eventSchedule) 추가** — 완성본 JSON-LD는 분석 원문 참조
5. **이미지 7.7MB(hero.jpg), 7.0MB, 6.7MB, 4.7MB...** — WebP 변환 + 리사이즈로 각 100~300KB 목표. LCP와 광고 랜딩 전환율 직접 타격

### 네이버 대응 (한국 시장 핵심)
- **네이버 서치어드바이저 등록** + sitemap 제출 (미등록 시 네이버 노출은 복권)
- **네이버 플레이스** 등록 검토 (고정 장소·시간의 오프라인 모임)
- **네이버 블로그 우선 발행**: "여의도 러닝 코스" 류 검색은 네이버 블로그 탭이 지배 — RELATED_ARTICLES 4개 글감을 자사보다 네이버 블로그에 먼저
- `public/llms.txt` 추가 (비용 0 — 서비스 정의·가격·환불을 마크다운으로)

### 사이트 구조 로드맵
Phase 0(즉시): 도메인 통일+robots+이미지+JSON-LD 정적화 → Phase 1(1~2주): /faq /reviews /pricing 라우트 분리+프리렌더 → Phase 2(1~2개월): /guide 콘텐츠 허브(Astro/Next SSG 검토) + /vs/running-crew 비교 페이지(AI 인용률 최고 유형)

## 4️⃣ 성장 / 유입 전략

### 유입 채널 Top 5 (효과×난이도)
1. **매칭 확정 공유카드 → 인스타 스토리 UGC 루프** (이미 구현된 shareCard.ts에 추천 코드만 추가 — CAC 0원)
2. **당근 동네생활·모임** (여의도/영등포/마포) — 지역 소모임 발견 채널 1순위, 무료
3. **인스타 릴스 — 목요일 세션 현장 기록** (매주 세션 = 콘텐츠 공장)
4. **블라인드 여의도 라운지** — 체험담 형태("모르는 사람들이랑 5km 뛰는 모임 다녀온 후기")
5. **오픈카톡 러닝방 씨딩** (`?apply=1` 원클릭 링크 활용)

### 성장 루프
```
신청 → 매칭 확정 (공유카드 ①) → 목요일 완주 → 완주 카드 발급 (공유 ②)
→ 인스타 태그 → 같은 정체성의 직장인이 발견 → 추천 코드 경유 신청 ③ → 반복
```
- **완주 카드 신설** — 매칭 카드보다 자랑 욕구가 훨씬 큼 (러닝 앱 기록 공유 심리)
- **세션 단체사진 의례** + 당일 밤 카톡 전달 + 태그 리워드 안내 (UGC의 90%는 요청해야 나옴)
- 정체성 언어: *"크루는 부담스럽지만 혼자는 못 뛰는 사람"*

### 추천 프로그램 개선
- 노출 시점을 신청 폼(전환 시점) → **완주 직후(만족 시점)** 카톡으로 재배치
- **shareCard에 추천 코드 굽기** + `?apply=1&ref=코드` 파라미터로 수동 입력 마찰 제거
- 티어 추가: 3명 성사 시 다음 4주 무료

### 기수제 리프레이밍
"4주 멤버십" → **"ONDO 여름 시즌 3기"** — 제품 변경 없이 매 4주마다 런칭 모멘텀 발생(트레바리 문법). D-14 예고(리드 DB에 우선 안내) → D-7 재등록 우선권+얼리버드 → D-3 실데이터 마감 임박 → 기수 종료 시 **시즌 리캡**("3기 N명, 총 O km" + 개인별 리캡 카드 = Spotify Wrapped 축소판). 겨울 시즌은 "윈터 챌린지" 프레임으로 계절 리스크 대응.

## 5️⃣ 리텐션 / 가격 분석

### 활성화 여정의 빈틈 (P0만)
| 빈틈 | 처방 |
|---|---|
| 입금이 최대 이탈 지점 (기한 없음, 리마인드 수동) | 기한 명시 + 복사 버튼 + **신청 즉시 알림톡 자동 발송** |
| 러닝 직후 골든아워(24h)를 안 씀 — 후기 요청 메시지가 notice.ts에 없음 | 당일 밤 후기+전환 오퍼 메시지 신설 |
| 체험→멤버십 전환 플로우 자체가 없음 | 48시간 한정 "체험비 5,000원 차감" 오퍼 |

### 핵심 신규 메시지 (M6 — 가장 중요)
> ○○님, 오늘 5km 완주 축하드려요! 🎉 오늘 함께 뛴 온도, 어땠나요? 한 줄 후기 남겨주시면 다음 매칭에 반영할게요.
> 그리고 — 오늘 낸 5,000원, 버리지 마세요. **48시간 안에 4주 멤버십 등록 시 체험비를 그대로 차감해 14,000원**에 시작할 수 있어요.

### 재등록 플로우 (현재 전무 — 4주 만료 = 자동 이탈)
- **3주차 러닝 직후** 조기 재등록 오퍼: "이번 주 안에 재등록하면 17,000원 + **지금 그룹 우선 유지**" (진짜 lock-in은 할인이 아니라 소셜 자산)
- **4주차 완주 직후** 개인화 리캡: "러닝 4회 · 20km · 만난 러너 12명. 한 달 전엔 '혼자서는 미루던' 분이었어요."
- 이탈 사유별 세이브 오퍼: 시간 없음→**이월(pause)**, 그룹 안 맞음→재매칭 보장, 가격→친구 추천 리마인드

### 가격 구조
- **앵커링이 거꾸로**: 체험 5,000원/회 vs 멤버십 4,750원/회 = 할인율 5%라 업그레이드 유인 없음 → **체험권 7,900원 인상 시 멤버십이 40% 할인으로 보임**. 인상이 부담이면 최소 `originalPrice`(코드에 있는데 미사용)로 취소선 앵커
- "회당 4,750원 · 커피 한 잔 값" 단위가격 표기 — 가장 싼 개선
- 검증 후 3단 구성: 체험 7,900 / **4주 19,000(추천)** / 8주 34,000(데코이)

### 알림톡 자동화 (주 30명 넘으면 수동 복붙 운영 병목)
- 알림톡(정보성, 동의 불필요): 접수·입금확인·D-1·당일·우천 / 친구톡·SMS(광고성, 동의 필요): 전환 오퍼·윈백
- notice.ts 함수들이 이미 템플릿 형태라 알림톡 템플릿 승인 신청에 거의 그대로 사용 가능
- Solapi/알리고 + Supabase Edge Function: INSERT→접수 알림, paid=true→확정, 수·목 18시 크론→리마인드
- ⚠️ 광고성 메시지는 (광고) 표기 + 수신거부 안내 + **21시~08시 발송 금지** — 러닝 직후 밤 10시 발송이면 정보성으로 구성하거나 익일 아침으로

## 6️⃣ 측정 / 리서치 / 포지셔닝

### 트래킹 수정 (GA4는 이미 살아있음 — G-7E1D5GER95)
1. page_view 이중 집계 제거 / 2. `lead_submit` 이벤트 신설 / 3. **`apply_step_completed {step}` 5단계 이벤트 추가** (폼 드롭오프 진단의 핵심) / 4. `payment_confirmed`는 어드민 입금확인 시점 기록 또는 주간 Supabase 대조 / 5. application_complete를 GA4 핵심 이벤트 지정
- **북극성 지표**: 주간 신청완료 수 + **재신청률**(같은 phone 2회차 — GA 아닌 Supabase가 진실의 원천)
- UTM 규칙: `?apply=1` 재진입 링크와 shareCard 링크에 자동 부착이 저비용 고효과

### 고객 리서치 — 이미 보유한 골드마인
- **리드 vs 신청 갭**: 전화번호는 남겼는데 신청 안 한 사람 = 인터뷰 1순위 ("번호까지 남기셨는데 그때 뭐가 걸리셨나요?")
- satisfaction_checks의 재참여 의향 X 응답 = 가장 값진 신호
- faq_open 이벤트의 question 속성 = 실제 불안 순위
- 인터뷰 질문 10종(JTBD 구조) 및 워터링홀(네이버 카페·블라인드·당근·소셜링 리뷰) 조사법은 분석 원문 참조

### 포지셔닝
- 이미 날카로움: 혼자 뛰기·대형 크루 대비는 FAQ에서 정면 공략 중
- **공백 1**: 소셜링/문토/프립(타겟이 가장 많이 비교할 대안) 대비 언급 0 → FAQ 추가: *"원데이 모임은 그날로 끝나지만, ONDO는 매주 목요일 반복되는 약속입니다. 루틴은 반복에서만 생깁니다."*
- **공백 2**: "페이스 매칭"이 주장뿐 — 매칭 성사율·재참여율 숫자를 증거로 노출
- 카테고리 프레임: "러닝크루"가 아니라 **"러닝 루틴 서비스"** 일관 유지 (크루로 불리는 순간 무료 대안과 가격 비교당함)

> **포지셔닝 스테이트먼트**: 혼자서는 러닝을 자꾸 미루게 되는 서울 2030 직장인에게, ONDO는 매주 목요일 저녁 여의도 한강에서 페이스가 맞는 3~5명과 달리는 러닝 메이트 매칭 서비스다. 일회성 소셜 모임과 달리 매주 반복되는 약속이라 러닝이 루틴이 되고, 대형 러닝크루와 달리 친목 부담 없이 뛰고 깔끔하게 헤어진다.
